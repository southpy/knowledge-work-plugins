# Agentic 反模式清单

对任何 Agent 设计进行评审时，逐条检查以下 8 条反模式。每条给出诊断信号和处置方法。

完整分析和案例见 `references/anti-patterns-detail.md`。

## 8 条反模式速查表

| # | 反模式 | 诊断信号 | 处置 |
|---|--------|----------|------|
| 1 | **系统提示当知识库** | System prompt > 1000 tokens，关键规则被遗忘 | 约定留提示，知识移 Skills |
| 2 | **工具数量失控** | Agent 频繁选错工具，>10 个工具 | 合并重叠工具，明确命名空间 |
| 3 | **验证闭环缺失** | Agent 宣告完成但无法验证结果 | 每类任务绑定验收标准（Harness） |
| 4 | **多 Agent 无边界** | 状态漂移，故障归因困难 | 明确角色权限，worktree 隔离 |
| 5 | **记忆不整合** | 长对话（>20轮）后决策质量下降 | 监控 token，超阈值自动触发压缩 |
| 6 | **没有评测** | 改了一个地方不知道有没有引入回归 | 失败案例立刻转测试用例 |
| 7 | **过早引入多 Agent** | 协调开销超过并行收益 | 先验证单 Agent 上限再扩展 |
| 8 | **约束靠期望不靠机制** | 规则在文档里，Agent 选择性遵守 | 改用工具验证 / Linter / Hook |

## 评审输出格式

对每条反模式输出：

```
### 反模式 N：[名称]
**状态**：✅ 无问题 / ⚠️ 风险 / ❌ 存在问题
**证据**：[在设计中发现的具体信号]
**建议**：[具体的改进动作]
```

最终给出综合风险评级：**低（0-2条问题）/ 中（3-4条）/ 高（5+条）**
