# /review-agent — Agent 架构评审

接收已有 Agent 设计（文档、描述或代码），对照规范清单逐条评审，输出结构化评审报告。

## 评审执行顺序

```
Step 1 - 理解设计
  - 提取：角色定义、工具链、上下文结构、验证机制、Agent 数量
  - 如果输入不完整，列出信息缺口（不猜测，不假设）

Step 2 - 8 条反模式逐条检查
  - 每条：判断是否存在该反模式的诊断信号
  - 给出具体证据（引用设计中的原文或结构）

Step 3 - 上下文结构审查（context-design 原则）
  - 五层分层是否清晰？
  - 常驻层 token 估算
  - 知识类内容是否已 Skill 化

Step 4 - 工具链审查（ACI 原则）
  - 工具数量
  - description 质量
  - 错误处理完整性

Step 5 - Harness 完整性检查
  - 验收标准是否可量化
  - 反馈信号来源是否为非 Agent 自报
  - 是否有回退机制

Step 6 - 综合评级 + 优先建议
```

## 评审行为规范

- 没有证据的问题不评审（信息缺口单独列出）
- 每个 ❌ 必须有具体引用或量化数据
- 建议必须是可执行的具体动作，不是"建议改善"
- 单 Agent 设计的 AP-4 和 AP-7 自动标记 ✅
